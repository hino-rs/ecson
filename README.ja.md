# Ecson 

> Rust向けの超高速でECS駆動なステートフル双方向サーバーフレームワーク

**Ecson**(エクソン)は**リアルタイム性とステート管理が重要な要素となるアプリケーション**のために構築された実験的なWebフレームワークです。`tokio`の非同期I/O処理能力と`bevy_ecs`の超高速なデータ志向設計を統合することで、`Arc<Mutex<T>>`のような同期プリミティブを一切使用することなく、マルチプレイヤーバックエンドやリアルタイムコラボレーションツール、空間シミュレーションなどを構築することが可能です。

## Web開発にECSを採用する理由

従来の非同期Webフレームワークは、ステートレスなCRUD APIの構築には最適です。しかし、2Dメタバースやライブホワイトボード、マルチプレイヤーゲームといった複雑なリアルタイムサーバーを構築する段階になると、必ず「グローバル共有状態」という壁にぶつかります。数千もの持続的なWebSocket接続間でロック機構を管理すると、デッドロックやスレッドのブロック、複雑に絡み合ったコードといった問題が発生してしまいます。

Ecsonはこの問題を解決するため、パラダイムを以下の様に転換します:

1. **接続はエンティティとして扱う：** WebSocket接続はすべて、グローバルECSワールド内でエンティティとして生成。
2. **データはコンポーネントとして管理：** ユーザーID、座標、ルーム名、体力値などは、エンティティに紐付けられた単なるコンポーネント。
3. **ロジックはシステムとして実装：** 必要なデータのみをクエリする純粋な関数型ロジック（システム）を書きます。ECSスケジューラは、競合が発生しないシステムを複数スレッドで並列実行します。**ロックは一切不要です。**

## ログ出力

Ecsonは`log`クレートを使用しています。ログを表示するには`env_logger`などをアプリ側で初期化してください。

## 仕様技術

このフレームワークは主にTokioとBevyという巨人の肩の上に立っています。
他にも多数のクレートに支えられているので、以下に並べます。

- tokio
- tokio-tungstenite
- bevy_ecs
- futures-util
- wtransport
